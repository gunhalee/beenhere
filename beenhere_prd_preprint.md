# PRD: 위치 기반 글 공유 서비스

## 1\. 제품 개요

### 1.1 제품 정의

사용자가 물리적 장소에 글을 남기고, 마음에 드는 글을 라이크하여 자신의 위치로 퍼뜨리는 위치 기반 글 공유 서비스.

### 1.2 핵심 컨셉

"이야기꾼들의 세계" — 가는 곳마다 게시판이 있고, 이야기꾼들이 글을 남기거나 다른 이야기꾼의 글을 옮겨 붙인다. 게시판에 글을 남긴 사람의 이야기보따리를 펼쳐볼 수 있다.

### 1.3 핵심 원칙

* 글은 장소에 묶여 있고, 사람은 글을 통해 발견된다.
* 유일한 상호작용은 라이크이며, 라이크는 항상 재공유를 수반한다.
* 댓글, 팔로우, 알림, 알고리즘은 존재하지 않는다.
* 서비스가 사용자를 부르지 않는다. 사용자가 스스로 찾아온다.

\---

## 2\. 사용자 경험 흐름

### 2.1 발견

앱을 열면 사용자의 현재 위치에서 가까운 순서로 글이 피드에 표시된다. 각 글에는 라이크 수, 원작성자, 직전 공유자가 표시된다.

### 2.2 탐색

원작성자 또는 직전 공유자의 프로필에 진입한다. 그 사람이 쓰고 모은 글들을 보고, 거기서 또 다른 사람의 프로필로 이어진다. 탐색에 깊이 제한은 없다.

### 2.3 전파

마음에 드는 글을 라이크하면 사용자의 현재 위치에 재공유된다. 주변 사람들의 피드에 이 글이 뜨고, 직전 공유자로 사용자의 이름이 붙는다.

### 2.4 확인

자신이 작성한 글의 라이크 수와 라이크한 사람 목록을 확인한다. 라이크한 사람의 프로필에 들어가 어떤 사람이 자신의 글에 공명했는지 살펴본다. 이 정보는 작성자 본인만 볼 수 있다.

## 3\. 기능 명세

### 3.1 글 작성 및 노출

|항목|내용|
|-|-|
|작성 조건|사용자는 현재 위치에서 글을 작성할 수 있다|
|좌표 고정|글은 작성 장소에 좌표가 고정된다|
|노출 범위|고정된 좌표로부터 10km 이내에 접속한 다른 사용자의 피드에 노출된다|
|자기 글 열람|자신이 작성한 글은 위치와 무관하게 항상 볼 수 있다|

### 3.2 라이크

|항목|내용|
|-|-|
|라이크 가능 위치|피드에서 또는 타인의 프로필에서 발견한 글을 라이크할 수 있다|
|재공유|라이크한 글은 라이크 시점의 사용자 위치에 새로운 좌표로 고정되어 재공유된다|
|노출 범위|새로 고정된 좌표 10km 이내에 접속한 다른 사용자의 피드에 노출된다|
|자기 라이크 열람|라이크한 글은 위치와 무관하게 자신의 프로필에서 항상 볼 수 있다|
|라이크 수 노출|라이크 수는 글을 보는 모든 사람에게 노출된다|
|조용한 소장|재공유 없이 소장만 하는 기능은 없다. 라이크는 항상 재공유를 수반한다|

**라이크의 복합 효과:** 라이크 한 번은 세 가지 결과를 동시에 만들며, 이 세 가지를 분리할 수 없다.

1. 글이 라이크한 사용자의 현재 위치에 재공유된다
2. 글의 노출 기간이 연장된다
3. 글이 라이크한 사용자의 프로필에 추가된다

### 3.3 글의 노출 기간

1. 글이 작성된 시점에서 한달동안 노출이 된다.

글이 라이크를 얻을 때마다 노출 기간이 한달 연장된다.

\## 3.4 피드

|항목|내용|
|-|-|
|정렬 기준|사용자의 현재 위치에서 가까운 순서로 정렬된다|
|글 표시 정보|라이크 수, 원작성자, 직전 공유자|
|직전 공유자 규칙|직전 공유자가 곧 원작성자인 경우 원작성자만 표시된다|
|전파 경로|전체 전파 경로는 피드에서 표시되지 않는다|

**거리 순 정렬의 효과:** 이동하면 피드 순서가 실시간으로 바뀐다. 사용자가 서 있는 바로 그 장소에서 작성되거나 라이크된 글이 가장 먼저 보인다.

### 3.5 작성자 전용 정보

|항목|내용|
|-|-|
|라이크한 사람 목록|자신이 작성한 글에 한해, 라이크한 사람의 목록을 볼 수 있다. 전파 경로의 트리 구조는 제시되지 않으며, 누가 라이크했는지의 최종 결과만 보여준다|
|프로필 진입|라이크한 사람의 프로필에 진입할 수 있다|
|접근 제한|이 정보는 작성자 본인만 볼 수 있다|

### 3.6 프로필과 탐색

|항목|내용|
|-|-|
|프로필 진입 조건|피드에서 글의 원작성자 또는 직전 공유자의 프로필에 진입할 수 있다. 라이크 없이 즉시 가능하다|
|프로필 내 정보|타인의 프로필에서는 그 사람이 작성한 글과 라이크한 글을 모두 볼 수 있다|
|탐색 깊이|타인의 프로필에서 발견한 글의 원작성자 또는 공유자의 프로필에 다시 진입할 수 있다. 깊이 제한은 없다|
|프로필 개방성|프로필 비공개, 잠금, 승인 절차는 존재하지 않는다|

\---

## 4\. 글의 다중 존재 구조

하나의 글은 여러 좌표에 동시에 존재할 수 있다.

* 원작성자가 고정한 좌표
* 라이크한 사람들이 각각 고정한 좌표

각 좌표는 독립적으로 10km 반경 피드에 노출된다. 동일한 글이라도 좌표에 따라 직전 공유자가 다르게 표시되며, 원작성자는 모든 좌표에서 항상 동일하게 표시된다.

\---

## 5\. 프로필 진입 경로

사용자 유형에 따라 타인의 프로필에 진입할 수 있는 경로가 다르다.

**일반 사용자의 진입 경로:**

* 피드의 글 → 원작성자 프로필
* 피드의 글 → 직전 공유자 프로필
* 타인 프로필 내 글 → 해당 글의 원작성자/공유자 프로필

**글 작성자의 추가 진입 경로:**

* 자신이 작성한 글 → 라이크한 사람 목록 → 해당 사람의 프로필

## 6\. 없는 기능

|기능|상태|
|-|-|
|댓글|없음|
|팔로우|없음|
|알림|없음. 사용자가 직접 앱을 열어 확인한다|
|알고리즘 추천|없음. 피드는 위치 기반 거리 순으로만 구성된다|
|조용한 소장|없음. 라이크는 항상 재공유를 수반한다|
|전파 경로 공개 표시|없음. 작성자 본인에게도 트리 구조가 아닌 라이크한 사람 목록만 제공된다|
|프로필 비공개|없음|
